Результатом стало створення відповідного документа для авторського проєкту — платформи гейміфікації віддаленої роботи під назвою "PackageDelivery", метою якої є підвищення мотиваційного рівня та активності співробітників в умовах дистанційної зайнятості.
     Початковий етап роботи передбачав дослідження проблематики, що виникає через зниження ефективності праці та емоційного залучення команд у віддаленому форматі. Також проведено огляд наявних корпоративних платформ з елементами геймдизайну, що дозволило виявити конкурентні переваги та унікальну цінність пропонованого рішення.
     Ключовим кроком стало формулювання концептуального бачення (Vision Statement) системи PackageDelivery, в якому визначено стратегічну мету платформи, її цільових користувачів та пріоритетні функціональні напрямки. Проєкт фокусується на впровадженні чіткого ігрового механізму оцінювання робочих результатів і стимулюванні командної співпраці.
     Детально описано бізнес-модель системи, яка охоплює взаємозв'язки між учасниками, подіями у системі, системою нагород та інструментами аналітики. Визначено чотири основні технологічні компоненти: бекенд-частину, IoT-модуль (NFC/RFID-ідентифікатори та LED-дисплеї для візуалізації), а також веб- та мобільні інтерфейси, що разом формують цілісну екосистему мотивації персоналу.
     Технологічні рішення включають серверну реалізацію на базі .NET Core із застосуванням СУБД Postgres та взаємодію з клієнтськими додатками через REST API. IoT-компонент забезпечує матеріальне відображення досягнень працівників, тоді як веб- та мобільні версії призначені для користувацької взаємодії та адміністративного управління.

1. Business Requirements
1.1 Background
     У сучасному світі електронної комерції та онлайн-шопінгу зростає потреба у безпечних, автоматизованих та зручних рішеннях для доставки посилок. Традиційні методи доставки "до дверей" мають численні обмеження: необхідність присутності одержувача вдома, ризик крадіжок посилок, залишених біля дверей, незручність для кур'єрів, які змушені здійснювати повторні спроби доставки, та високі операційні витрати логістичних компаній.
     За даними міжнародних досліджень логістичної галузі, до тридцяти відсотків спроб доставки посилок є невдалими через відсутність одержувача за адресою. Це призводить до збільшення витрат на повторну доставку, затримок у ланцюгу постачання та зниження задоволеності клієнтів. У міських умовах, де ритм життя є високим, а графіки роботи різноманітні, проблема стає особливо гострою. Працівники, які працюють у стандартному офісному режимі, часто не можуть бути вдома в години, коли кур'єри здійснюють доставку, що створює замкнене коло незручностей як для одержувачів, так і для служб доставки.
     Пандемія COVID-19 додатково прискорила перехід до безконтактних методів доставки та отримання товарів. Споживачі все частіше віддають перевагу рішенням, які дозволяють отримувати посилки у зручний для них час без необхідності особистого контакту з кур'єром або персоналом пунктів видачі. Ця тенденція зберігається і після пандемії, оскільки користувачі звикли до зручності та безпеки безконтактних рішень.
     Існуючі рішення, такі як традиційні поштові відділення або пункти видачі, часто мають обмежений графік роботи, черги та необхідність пред'явлення документів. Автоматизовані поштомати, хоча й вирішують частину проблем, часто використовують складні системи ідентифікації з одноразовими кодами або SMS-повідомленнями, що створює незручності для користувачів та додаткові операційні витрати. Необхідність запам'ятовувати або зберігати коди доступу, залежність від якості мобільного зв'язку для отримання SMS, ризик втрати паперового повідомлення з кодом — усе це знижує загальну зручність використання існуючих систем.
     У цьому контексті технологія NFC пропонує елегантне рішення для ідентифікації користувачів. Вона забезпечує швидку, безпечну та зручну автентифікацію за допомогою смартфонів або спеціальних карток, які більшість користувачів вже мають при собі. NFC-технологія широко інтегрована в сучасні мобільні пристрої та активно використовується для безконтактних платежів, доступу до приміщень та інших сценаріїв автентифікації, що робить її знайомою та зрозумілою для широкої аудиторії.
     Крім того, логістичні компанії стикаються з необхідністю оптимізації використання простору для зберігання посилок. Різноманітність розмірів посилок створює проблему ефективного розміщення у комірках обмеженої місткості. Неефективне використання доступного простору призводить до ситуацій, коли поштомат фізично не може прийняти нові посилки, навіть якщо загальний об'єм вільного місця є достатнім. Це знижує пропускну здатність системи та негативно впливає на економічну ефективність бізнесу.
     Система поштових шафок з NFC-доступом розробляється саме для комплексного вирішення цих викликів. Вона поєднує зручність цілодобового доступу, надійність автоматизованої ідентифікації, безпеку зберігання посилок та інтелектуальну оптимізацію використання простору. Впровадження математичних алгоритмів для оптимального розміщення посилок дозволяє максимізувати ефективність використання кожної комірки, а інтеграція з Backend-системою забезпечує повний контроль над процесами доставки та отримання з можливістю аналітики та прогнозування.
1.2 Business Opportunity
     Ринок автоматизованих рішень для доставки посилок демонструє стійке зростання в усьому світі. За прогнозами аналітичних агентств, світовий ринок смарт-локерів досягне значних обсягів у найближчі роки, що зумовлено кількома ключовими факторами. По-перше, це експоненційне зростання електронної комерції, яке прискорилося під час пандемії та продовжує демонструвати високі темпи. По-друге, урбанізація та ущільнення міської забудови створюють попит на компактні рішення для доставки останньої милі. По-третє, зміна споживчих очікувань щодо зручності та швидкості доставки змушує логістичні компанії шукати нові технологічні рішення.
     Цільовими сегментами для впровадження системи є житлові комплекси та багатоквартирні будинки, де проблема пропущених доставок є найбільш гострою. Мешканці сучасних житлових комплексів цінують зручність отримання посилок без необхідності координації з кур'єром та безпеку зберігання до моменту забору. Офісні центри та бізнес-парки також представляють значний потенційний ринок, оскільки централізована обробка особистих посилок співробітників дозволяє зменшити навантаження на reception та підвищити загальну організацію робочого простору.
     Торгові центри та транспортні вузли є природними локаціями для розміщення поштоматів завдяки високій прохідності та зручності доступу для покупців інтернет-магазинів. Університетські кампуси представляють окремий сегмент, де студенти та персонал активно використовують онлайн-шопінг, а інтеграція з існуючими університетськими NFC-картками робить впровадження особливо зручним. Логістичні компанії та сервіси доставки розглядають автоматизовані поштомати як інструмент оптимізації маршрутів кур'єрів та зменшення витрат на повторні доставки.
     На ринку існують рішення від таких компаній, як InPost, Amazon Hub Locker, Luxer One, Нова Пошта та інших. InPost є лідером європейського ринку з понад двадцятьма тисячами поштоматів, але використовує SMS-коди та мобільний додаток для доступу, що створює залежність від якості мобільного зв'язку. Amazon Hub Locker інтегрований з екосистемою Amazon, але є закритою системою, доступною лише для замовлень з цього маркетплейсу. Luxer One фокусується на житлових комплексах, але має високу вартість впровадження та складність інтеграції. Нова Пошта має широку мережу поштоматів по Україні, але використовує штрих-коди та QR-коди, що потребує роздрукованих документів або завантаження додатку.
     Конкурентний аналіз показує, що більшість існуючих продуктів або обмежуються базовими функціями доступу через коди, або функціонують як частина великих логістичних мереж, що вимагають значних ресурсів для налаштування та підтримки.
     Порівняльний аналіз свідчить, що пропонована система займає унікальну нішу між простими кодовими рішеннями та комплексними корпоративними платформами. Існуючі рішення орієнтовані насамперед на великі організації з розвиненою інформаційною інфраструктурою та значними ресурсами для впровадження. На противагу цьому, система поштових шафок з NFC-доступом зосереджується на автономності, простоті інсталяції та ефективності використання без необхідності залучення складних інтеграцій.
     Ключовою відмінністю системи є використання NFC-технології для миттєвої автентифікації, що робить процес доступу максимально простим та інтуїтивним. Користувачу достатньо ввести свій NFC-код через клавіатуру IoT-пристрою, і комірка відкривається автоматично без необхідності сканувати QR-коди або чекати на SMS.
     Інтелектуальний алгоритм розміщення посилок є унікальною особливістю системи. Він аналізує тривимірні параметри кожної посилки, автоматично визначає оптимальну орієнтацію для розміщення та обирає комірку, яка забезпечує найкращий коефіцієнт використання простору. Це дозволяє розмістити більше посилок у тому самому фізичному об'ємі поштомату порівняно з простими системами першої вільної комірки.
     Система забезпечує повну автономність роботи завдяки локальній базі даних комірок в IoT-пристрої. Навіть при тимчасовій втраті з'єднання з Backend-сервером, поштомат продовжує функціонувати, обслуговуючи користувачів на основі локально збереженої інформації. Після відновлення зв'язку відбувається автоматична синхронізація всіх операцій.
     Низька вартість впровадження досягається за рахунок використання відкритої архітектури на базі доступних компонентів. Мікроконтролер ESP32 коштує лише кілька доларів, але надає всю необхідну функціональність включно з WiFi-модулем. Відсутність ліцензійних платежів за програмне забезпечення, оскільки всі використовувані технології є open-source, значно знижує операційні витрати.
     Таким чином, система поштових шафок з NFC-доступом формує збалансоване середовище для автоматизації процесів доставки, орієнтоване на малий і середній бізнес, управляючі компанії житлових комплексів, операторів офісних центрів та освітні заклади. Система забезпечує необхідний рівень функціональності для побудови ефективної інфраструктури доставки без надмірної складності у розгортанні та обслуговуванні.
2. Vision of the Solution
2.1 Vision Statement
     Система поштових шафок з NFC-доступом має на меті кардинально змінити підхід до доставки та отримання посилок, зробивши цей процес максимально зручним, безпечним та ефективним для всіх учасників. Концепція проєкту базується на ідеї створення повністю автоматизованої інфраструктури, яка працює цілодобово без необхідності присутності персоналу, забезпечуючи при цьому високий рівень безпеки та контролю за кожною операцією.
     Основне призначення системи полягає у формуванні зручного та доступного способу отримання посилок, де клієнти мають свободу забирати свої замовлення у будь-який зручний час без прив'язки до графіків роботи поштових відділень або необхідності очікування кур'єра вдома. Система передбачає перетворення традиційного процесу доставки у безшовну, автоматизовану операцію з миттєвою ідентифікацією через NFC-технологію.
     Передбачається, що використання системи сприятиме підвищенню задоволеності клієнтів завдяки зручності та доступності, оптимізації роботи кур'єрів через зменшення повторних візитів та зниженню операційних витрат логістичних компаній. У довгостроковій перспективі система має стати важливим елементом інфраструктури міської логістики, де кожен житловий комплекс, офісний центр та торговий центр має власний поштомат.

Програмна система виконує такі основні завдання:
* Забезпечує цілодобовий безконтактний доступ до посилок через NFC-ідентифікацію
* Створює зручний процес доставки для кур'єрів з інтелектуальними підказками щодо розміщення посилок
* Оптимізує використання простору комірок через математичні алгоритми розміщення
* Підтримує email-сповіщення для інформування користувачів про статус доставки
* Забезпечує безпечне зберігання посилок з повною фіксацією всіх операцій
Система створює цінність для всіх рівнів користувачів:
* Клієнти отримують можливість забирати посилки у зручний для них час, без черг та очікування, з миттєвим доступом через смартфон
* Кур'єри отримують інструмент для швидкого та ефективного розміщення посилок з автоматичним вибором оптимальної комірки та фіксацією всіх операцій
* Логістичні компанії отримують зменшення витрат на повторні доставки, повний контроль над процесами та аналітику для прийняття рішень
* Управляючі компанії житлових комплексів та офісних центрів отримують додаткову цінність для мешканців та підвищення престижу об'єкту
     Загалом, призначення системи полягає у створенні середовища, у якому отримання посилок стає простим та зручним процесом, а доставка перестає залежати від координації графіків користувача та кур'єра. У ширшому контексті система розглядається не лише як інструмент для автоматизації доставки, а як елемент стратегічної трансформації міської логістики.
2.2 Major Features
     Система складається з трьох основних компонентів, які працюють в тісній інтеграції для забезпечення повного циклу доставки та отримання посилок.
Серверна частина
     Серверна частина реалізована на платформі ASP.NET Core з використанням Entity Framework Core для роботи з базою даних PostgreSQL. Вона виконує роль центрального координатора всіх процесів, забезпечуючи надійне зберігання даних, обробку бізнес-логіки та координацію взаємодії між IoT-пристроями та майбутніми клієнтськими додатками.
     Backend надає RESTful API для всіх операцій системи. Модуль управління користувачами дозволяє реєструвати нових клієнтів та кур'єрів, генерувати унікальні QR-коди для прив'язки NFC-пристроїв та валідувати NFC-коди при доступі до поштоматів. Система автоматично визначає роль користувача та надає відповідні права доступу.
     Управління посилками включає повний життєвий цикл від створення до отримання клієнтом з автоматичним управлінням статусами. Кур'єри можуть отримувати списки посилок для доставки, змінювати статуси та фіксувати розміщення у комірках. Клієнти можуть переглядати свої посилки та отримувати інформацію про готовність до забору.
     Email-нотифікації автоматично інформують користувачів про важливі події: підтвердження реєстрації, доставку посилки у поштомат з номером комірки, підтвердження отримання. Всі повідомлення відправляються асинхронно без блокування основних операцій.
     Модуль аналітики використовує математичні методи для прогнозування часу доставки на основі історичних даних, статистичного аналізу ефективності системи та виявлення патернів активності користувачів. Адміністративний інтерфейс надає повний CRUD для всіх сутностей системи, експорт даних у різних форматах та інструменти для моніторингу роботи поштоматів.
IoT-клієнт
     IoT-клієнт реалізований на мікроконтролері ESP32 з MicroPython та включає LCD дисплей для відображення інформації, матричну клавіатуру для введення NFC-кодів, LED індикатори для візуального підтвердження операцій та електромагнітне реле для керування замками комірок.
     Архітектура побудована на основі машини станів, яка забезпечує чітку логіку переходів між режимами роботи. Користувач може обрати режим кур'єра для доставки посилок або режим клієнта для отримання, ввести свій NFC-код через клавіатуру, після чого система валідує його через Backend API та виконує відповідні операції.
     Для кур'єрів IoT-клієнт запитує список посилок для доставки та для кожної посилки викликає алгоритм оптимального розміщення. Алгоритм аналізує тривимірні розміри посилки та поточний стан комірок, обираючи ту, що забезпечує найкращий коефіцієнт використання простору. Система відображає рекомендацію на LCD, відкриває обрану комірку та фіксує операцію після підтвердження кур'єра.
     Для клієнтів система запитує список комірок з їх посилками та послідовно відкриває кожну, очікуючи підтвердження отримання. Після завершення всі комірки очищаються у локальній базі даних та стають доступними для нових посилок.
     Математичний алгоритм оптимального розміщення є ключовою інновацією системи. Він обчислює об'єм посилки, фільтрує доступні комірки, виконує тривимірну перевірку розмірів з автоматичною ротацією посилки для знаходження оптимальної орієнтації, обчислює коефіцієнт використання та оцінку ефективності за спеціальною функцією, що враховує баланс між недостатнім та надмірним заповненням.
     Локальна база даних зберігає інформацію про шість комірок різних розмірів у оперативній пам'яті пристрою. Система автоматично оновлює стан комірок при розміщенні та очищенні посилок, забезпечуючи актуальність даних для алгоритму оптимізації.
     Офлайн-режим дозволяє продовжувати роботу при тимчасовій втраті з'єднання з Backend завдяки локальним даним. Після відновлення зв'язку відбувається автоматична синхронізація всіх операцій.
Клієнтські додатки (майбутні релізи)
     Веб-клієнт на React та TypeScript надасть повний функціонал для всіх типів користувачів: особистий кабінет для клієнтів з переглядом посилок та статусів, інструменти для кур'єрів з маршрутизацією доставок, адміністративну панель з візуалізацією даних та управлінням системою.
     Мобільний клієнт на Flutter забезпечить швидкий доступ до статусу посилок, push-сповіщення про важливі події, вбудований сканер QR-кодів для прив'язки NFC та можливість роботи в офлайн-режимі з автоматичною синхронізацією.
3. Scope and Limitations
3.1 Scope of Initial Release
     Початковий реліз системи фокусується на створенні повнофункціонального ядра, яке забезпечує всі критичні операції для роботи поштоматів з NFC-доступом.
     Серверна частина включає повний REST API з документацією Swagger для легкої інтеграції, модуль автентифікації на основі JWT-токенів, управління користувачами з реєстрацією клієнтів та кур'єрів, генерацію QR-кодів для прив'язки NFC-пристроїв, валідацію NFC-кодів з визначенням ролі користувача, управління посилками з повним життєвим циклом через статуси Pending, In Progress, Delivered та Received, email-нотифікації для інформування про ключові події, модуль аналітики з прогнозуванням часу доставки та статистичним аналізом, адміністративний модуль з CRUD операціями для всіх сутностей, експортом даних у JSON та CSV форматах, очищенням старих даних та переглядом статистики системи.
     База даних PostgreSQL організована у вигляді семи таблиць: Users для зберігання облікових записів з email-адресами та NFC-кодами, Roles для визначення ролей Client, Courier та Administrator, UserRoles для реалізації відношення багато-до-багатьох, Packages для зберігання інформації про посилки з тривимірними параметрами та статусами, PackageCategories для категорій посилок, DeliveryStatuses для статусів процесу доставки, AppLogs для централізованого логування. Entity Framework Core забезпечує міграції для версіювання схеми та конфігурацію зв'язків через Fluent API.
     IoT-клієнт на ESP32 з MicroPython включає апаратну периферію: LCD дисплей шістнадцять на два символи, матричну клавіатуру чотири на чотири, два LED індикатори, електромагнітне реле для замків, WiFi модуль для зв'язку з Backend. Програмне забезпечення реалізує машину станів з сімома станами для обробки різних сценаріїв взаємодії, режим кур'єра з алгоритмом оптимального розміщення посилок, режим клієнта для отримання посилок з автоматичним відкриттям комірок, математичний алгоритм з тривимірною перевіркою розмірів та оцінкою ефективності, локальну базу даних шести комірок різних розмірів, систему статистики для моніторингу роботи пристрою, підтримку локалізації українською та англійською мовами, офлайн-режим з буферизацією операцій для синхронізації.
3.2 Scope of Subsequent Releases
     Наступні релізи будуть поступово розширювати функціональність системи додаючи користувацькі інтерфейси та розширені можливості.
     Веб-клієнт на React та TypeScript надасть особистий кабінет для клієнтів з переглядом всіх посилок та їх статусів у реальному часі, історією доставок з фільтрацією, прогнозом часу доставки, сповіщеннями про зміну статусу, можливістю завантаження QR-коду для прив'язки NFC. Для кур'єрів буде доступний список посилок у статусі Pending з інструментами для маршрутизації, можливість зміни статусу на In Progress, статистика доставлених посилок. Адміністратори отримають дашборд з візуалізацією даних, інструменти для управління користувачами та посилками, експорт даних у різних форматах, перегляд та очищення логів, налаштування системної конфігурації.
     Мобільний клієнт на Flutter забезпечить швидкий доступ до статусу посилок, push-сповіщення про доставку та готовність до отримання, вбудований сканер QR-кодів, можливість емуляції NFC-картки на підтримуваних пристроях, карту локацій поштоматів з прокладанням маршруту, історію транзакцій, офлайн-режим з локальним кешуванням.
     Розширення IoT-функціональності включатиме автоматичне зчитування NFC без ручного введення, підтримку RFID-карток, біометричну автентифікацію, датчики відкриття та закриття комірок, датчики ваги для контролю вилучення посилок, інтеграцію з системами відеоспостереження.
     Розширена аналітика додасть модуль машинного навчання для прогнозування попиту на комірки, кластерний аналіз поведінки користувачів, рекомендаційну систему для оптимізації розміщення поштоматів, виявлення аномалій та шахрайства, експорт звітів у PDF та Excel.
     Масштабування системи передбачає підтримку мережі поштоматів у різних локаціях, міжлокаційну маршрутизацію посилок, інтеграцію з платіжними системами, API для інтеграції з інтернет-магазинами, підтримку міжнародних доставок, багатоорендну архітектуру для обслуговування кількох організацій.
4. Business Context
4.3 Operating Environment
     Система розробляється з використанням сучасних технологій, що забезпечують надійність та масштабованість.
     Серверна частина реалізована мовою C# на платформі .NET десятої версії, що забезпечує високу продуктивність та кросплатформенність. Для зберігання даних використовується PostgreSQL — надійна реляційна СУБД з підтримкою складних запитів та інтеграцією через Entity Framework Core. Взаємодія відбувається через HTTPS з передачею даних у форматі JSON. Безпека забезпечується JWT-токенами для автентифікації та TLS-шифруванням для всіх з'єднань.
     Розгортання можливе як у локальному середовищі для повного контролю над інфраструктурою, так і у хмарних сервісах Azure для простоти масштабування. Середовище розробки або JetBrains Rider з інтеграцією Git для контролю версій. Використовується стратегія GitFlow для управління релізами та стабільності основної гілки.
     IoT-клієнт базується на мікроконтролері ESP32 з підтримкою WiFi та достатньою обчислювальною потужністю для виконання алгоритму оптимізації. MicroPython обраний як мова програмування завдяки простоті розробки та підтримці необхідних бібліотек. Апаратна периферія включає LCD дисплей для інформування користувачів, матричну клавіатуру для введення даних, LED індикатори для візуального підтвердження, електромагнітне реле для керування замками.
     Веб-клієнт розроблятиметься на React з TypeScript для типобезпеки та Material-UI для сучасного інтерфейсу. Мобільний клієнт на Flutter дозволить створити єдину кодову базу для Android та iOS. 
     Обидва клієнти взаємодіятимуть з Backend через REST API з JWT-авторизацією.
     Безпека системи забезпечується на всіх рівнях: HTTPS для всіх з'єднань.
     Система підтримує локалізацію для української та англійської мов з можливістю легкого додавання нових перекладів. Всі текстові рядки винесені у словники, що спрощує підтримку та розширення.
